# Albums

Соцсеть, где единица контента — **альбом**: курируемая коллекция фото, видео и голосовых
заметок, которая читается как история. Плюс вторая поверхность — инста-стайл лента постов
(пост = фото/видео или карусель).

Прод: https://poklontsevv.github.io/albums/

## Стек

Статика без сборки (HTML + ES-модули) на GitHub Pages, данные — Supabase
(Postgres + RLS, Google-auth, Storage, SECURITY DEFINER RPC). Архитектура: [ARCHITECTURE.md](ARCHITECTURE.md).

## Страницы

| Файл | Что это |
|---|---|
| `index.html` | Лента рекомендаций альбомов + поиск (`?q=`) |
| `album.html?id=` | Альбом: обложка, главы, медиа, голосовые заметки, комментарии |
| `posts.html` | Инста-лента постов + композер карусели |
| `profile.html?u=` | Профиль, дружба, сетка альбомов |
| `editor.html?id=` | Создание/редактирование альбома |
| `friends.html` | Заявки, друзья, поиск людей |

## Приватность

Альбом виден: всем (`public`) / только автору (`private`) / друзьям (`friends`) /
друзьям кроме выбранных (`friends` + `album_exceptions`). Всё считает **одна** функция
`can_view_album()`, на которую опираются RLS-политики, политики Storage и фид-RPC.
Медиа лежат в приватном бакете и отдаются подписанными URL — приватный альбом
нельзя открыть по угаданной ссылке.

## Локальная разработка

```
node serve.js          # http://localhost:5085
```

## Миграции

Схема живёт в `supabase/migrations/`. Применение (нужен `.supabase-token` в корне,
он в `.gitignore`):

```
powershell tools\sql.ps1 -Path supabase\migrations\001_init.sql
```
