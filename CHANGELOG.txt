v0.1.0 — 2026-07-19
  Первая версия.
  База: схема v1 (13 таблиц, RLS default-deny, 32 политики, 24 функции),
        бакеты avatars (public) и media (private), Google-auth.
  Приватность: public / private / friends / friends-кроме-выбранных через
        can_view_album(); проверено 29 SQL-тестами (матрица видимости,
        enforcement RLS по ролям, фильтрация в SECURITY DEFINER RPC).
  Сайт: лента альбомов с рекомендациями и бесконечной прокруткой, поиск,
        страница альбома (главы, лайтбокс, голосовые заметки с волноформой),
        редактор альбомов (загрузка медиа, главы, обложка, видимость),
        инста-лента постов с каруселью и композером, профиль с
        редактированием, друзья (заявки/подтверждение/удаление),
        комментарии с ветками для альбомов и постов.

v0.9.0 — 2026-07-21
  Медиа переехало на Cloudflare R2 (egress $0), приватность прежняя:
        edge-функция r2-sign выдаёт временные ссылки после той же проверки
        can_view_media (двойной рубеж сохранён); старое медиа продолжает
        читаться из Supabase Storage — различение по префиксу r2/ в
        storage_path. Аватары не тронуты. Откат — флаг MEDIA_BACKEND в upload.js.
  Загрузка: серверная квота-леджер, ключ строит сервер (запись только в свой
        префикс), Content-Type пришпилен подписью; фото ≤2048px WebP q0.80,
        H.264-видео крупнее 1080p ужимается; починен MIME озвучки (был orig.bin).
  Безопасность: закрыта дыра — серверные функции модерации mod_* оказались
        доступны анониму через PostgREST в обход пароля (миграция 020,
        явный revoke на каждую функцию).
  Домен: albums.ink.

v0.11.0 — 2026-07-22
  «Слушать рассказ» стал полноэкранным слайдшоу: кадры сменяются по меткам
        дорожки автоматически, внизу плеер (пауза/плей, время, полоса
        прокрутки с точками-метками), вручную листать нельзя; видео в показе
        идёт без своего звука. Пробел — пауза, Escape/✕ — выход.
        Старая подсветка фото в сетке убрана.
  Режим «История» приведён к макету: медиа во всю ширину колонки одно под
        другим без обрезки, подпись под кадром; сначала грузится превью,
        оригинал подменяется по готовности. «Сетка» не менялась.
  Лайтбокс: подпись больше не накладывается на кнопки (вынесена из
        абсолютного позиционирования в поток), счётчик «N / M» обновляется
        при листании, на мобильных вернулись кнопки листания и закрытия
        (их прятало правило карусели постов).
  Обложки: белая плашка стала полупрозрачной с размытием фона, текст
        уменьшен на десктопе (23→19px) и вдвое на мобильных (13px),
        плашка компактнее и меньше закрывает медиа.
  Dev: serve.js читает порт из PORT, autoPort в launch.json.

v0.12.0 — 2026-07-22
  Статистика. Событийный лог stat_events: показ карточки в ленте, посещение
        альбома, время в альбоме (чекпойнтами, а не одной отправкой при
        закрытии — браузеры её теряют), переход по кнопке профиля. Гео берётся
        из часового пояса браузера, IP не пишем вообще.
  Кабинет автора (stats.html): показы, посещения, открываемость, удержание,
        зрители, разбивка по дням, странам, возрасту, полу и источникам
        перехода, таблица по альбомам. Читается одной definer-функцией
        stats_summary — сырые события клиенту недоступны.
  Демография: необязательные год рождения и пол в профиле (Google их при
        обычном входе не отдаёт). Колонки закрыты от прямого чтения через
        PostgREST — участвуют только в агрегатах.
  Кнопки Pro: до шести кнопок-ссылок в профиле, переходы считаются и видны
        в кабинете. Тариф хранится в profiles.plan; триггер не даёт выдать
        его себе самому запросом к своей строке (заодно закрыт самобан).
  Панель модерации: вкладка со статистикой продукта (пользователи, контент,
        активность по дням, страны, топ альбомов) и выдача Pro вручную —
        оплата пока не автоматическая.
  Тариф Pro на деле: фото до 4K вместо 2048px, файл до 500 МБ вместо 50,
        квота 100 ГБ; на бесплатном квота поднята до 1 ГБ. Обещание
        «оригиналы» с витрины убрано — оригиналы не храним.
  ПОЧИНЕНО: просмотры альбомов не считались вообще — вызов supabase-js без
        await не отправляет запрос. По той же причине не работал вход в
        панель модерации: mod-api не пропускал заголовки apikey и
        authorization в предполётной проверке CORS.
